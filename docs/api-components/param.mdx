## ParamField Component

The `<ParamField>` component is used to display the documentation for individual parameters

<CodeGroup>

```jsx Path
<ParamField path="user_id" type="string">
  The id of the user
</ParamField>
```

```jsx Query
<ParamField query="filter" type="string" default="none" required>
  The filtering command used to sort through the users
</ParamField>
```

```jsx Body
<ParamField body="user_age" type="integer" default="0" required>
  The age of the user. Cannot be less than 0
</ParamField>
```

</CodeGroup>

### Properties

| Name                    | Type    | Description                                                         |
| ----------------------- | ------- | ------------------------------------------------------------------- |
| `query`, `path`, `body` | string  | Whether it is a query, path, or body parameter followed by the name |
| `type`                  | string  | Expected type of the parameter's value                              |
| `required`              | boolean | Indicate whether the parameter is required                          |
| `default`               | string  | Default value                                                       |
| `placeholder`           | string  | Placeholder value for the input in the playground                   |
| `children`              | string  | Description of the parameter (markdown enabled)                     |

<Info>

All `Param` components are added as interactive elements in the API playground. See the [demo](./demo) for an example

</Info>
